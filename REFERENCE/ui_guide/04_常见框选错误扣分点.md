# 常见框选错误扣分点

本文档列举 8 种最常见的框选错误类型，每种附有示例说明、扣分标准和修正建议。

---

## 错误类型 1：框选偏移

### 错误描述
框选区域偏离了目标元素的实际位置，导致点击/输入操作发生在错误位置。

### 示例

```
正确框选：      [用户名输入框]    <- 框选区域与 input 吻合
偏移框选：   [用户名输入框]        <- 框选区域偏向左侧
              [用户名输入框]    <- 框选区域偏向右侧
```

### 扣分标准

| 偏移程度 | 偏移量 | 扣分 | 功能影响 |
|----------|--------|------|----------|
| 轻微偏移 | < 5px | 扣 1-2 分 | 基本无影响 |
| 中等偏移 | 5-15px | 扣 3-5 分 | 偶发点击失败 |
| 严重偏移 | 15-30px | 扣 6-10 分 | 经常点击到其他元素 |
| 完全偏离 | > 30px | 扣 10-15 分 | 点击到页面空白区域 |

### 修正建议
1. 重新拾取元素，确保选中目标元素正中央
2. 使用 CSS 选择器定位替代手动框选
3. 放大页面后再进行框选，提高精度

---

## 错误类型 2：框选范围过大

### 错误描述
框选区域远大于目标元素，包含大量无关区域。

### 示例

```
目标元素：[登录按钮]
正确框选：[登录按钮]
过大框选：[  登录按钮  ]  <- 包含了按钮周围的空白
更糟情况：[  登录按钮    取消按钮  ]  <- 包含了相邻按钮
```

### 扣分标准

| 超出范围 | 超出比例 | 扣分 | 功能影响 |
|----------|---------|------|----------|
| 略大 | < 20% | 扣 1-2 分 | 不影响功能 |
| 中等 | 20-50% | 扣 3-5 分 | 可能触发意外点击 |
| 较大 | 50-100% | 扣 6-8 分 | 易触发错误操作 |
| 远超大 | > 100% | 扣 8-12 分 | 经常触发错误操作 |

### 修正建议
1. 缩小框选范围，紧贴目标元素边缘
2. 对于按钮，只选 button 元素本身，不包含外层容器
3. 使用更精确的 CSS 选择器替代手动区域选择

---

## 错误类型 3：框选范围过小

### 错误描述
框选区域只覆盖了目标元素的一部分，导致操作不完整。

### 示例

```
目标元素：[用户名输入框|                  ]  <- 完整的输入框
过小框选：[用户名]  <- 只框选了 label 部分
或
目标元素：完整表格 A1:F10
过小框选：A1:C5     <- 只框选了表格左上角部分
```

### 扣分标准

| 覆盖程度 | 覆盖比例 | 扣分 | 功能影响 |
|----------|---------|------|----------|
| 覆盖大部分 | > 80% | 扣 1-3 分 | 功能基本正常 |
| 覆盖一半 | 50-80% | 扣 4-7 分 | 操作不完整 |
| 覆盖小部分 | 20-50% | 扣 8-12 分 | 操作失败率较高 |
| 几乎未覆盖 | < 20% | 扣 12-18 分 | 操作完全失败 |

### 修正建议
1. 扩大框选范围，确保完整覆盖目标元素
2. 对于输入框，确认包含了可输入区域
3. 对于表格，确认包含所有需要的行列
4. 使用浏览器开发者工具确认元素完整大小

---

## 错误类型 4：选错元素

### 错误描述
框选了错误的元素类型，例如把 label 当成 input 选中，或把 div 当成 button 选中。

### 示例

```
正确：选中 [button#loginBtn]          <- 登录按钮
错误：选中 [div.login-container]      <- 登录按钮外面的容器

正确：选中 [input#username]           <- 用户名输入框
错误：选中 [label[for="username"]]    <- 用户名标签
```

### 扣分标准

| 错误等级 | 错误说明 | 扣分 | 功能影响 |
|----------|---------|------|----------|
| 同类型不同实例 | 选错了同类型的其他元素 | 扣 8-12 分 | 操作到错误对象 |
| 不同类型 | 完全选错了元素类型 | 扣 12-15 分 | 操作无法执行 |
| 父子层级 | 选中了父容器而非目标子元素 | 扣 10-14 分 | 无法精准操作 |
| 兄弟层级 | 选中了相邻兄弟元素 | 扣 8-12 分 | 操作到相邻元素 |

### 修正建议
1. 在 RPA Studio 中高亮预览确认选中的元素
2. 使用 HTML 结构检查确认元素类型
3. 使用更精确的选择器：`button#loginBtn` 而不是 `button`
4. 确认元素在 DOM 树中的位置

---

## 错误类型 5：漏选

### 错误描述
必需的框选操作被遗漏，或者框选没有覆盖所有需要操作的页面区域。

### 示例

```
完整流程需要框选：
  1. 用户名输入框 ✓
  2. 密码输入框 ✓
  3. 登录按钮 ✓
  4. Excel 数据区域 × <- 漏选了
  5. 退出按钮 ×        <- 漏选了

结果：无法读取数据，无法退出登录
```

### 扣分标准

| 漏选类型 | 漏选对象 | 扣分 | 功能影响 |
|----------|---------|------|----------|
| 漏选必需元素 | 核心操作元素（登录/提交/数据） | 扣 15-20 分 | 功能严重缺失 |
| 漏选辅助元素 | 非核心但需要的元素（退出/确认） | 扣 8-12 分 | 部分功能缺失 |
| 漏选次数 | 每漏选 1 处额外扣分 | 扣 5-10 分/处 | 累积影响 |

### 修正建议
1. 对照作业要求清单逐项检查框选完整性
2. 运行作业流程检查是否存在未定义的步骤
3. 参考 `select_record.md` 记录所有框选操作
4. 对比示例工程确认有无遗漏步骤

---

## 错误类型 6：多选

### 错误描述
框选了不需要操作的无关元素，导致不必要的操作或误操作。

### 示例

```
需要的操作：只框选"登录"按钮
多选情况：同时框选了"登录"和"取消"按钮
结果：点击时可能点到"取消"按钮，关闭登录弹窗
```

### 扣分标准

| 多选类型 | 多选对象 | 扣分 | 功能影响 |
|----------|---------|------|----------|
| 多选相邻元素 | 相邻的无关按钮/链接 | 扣 5-8 分 | 可能触发意外功能 |
| 多选空白区域 | 目标元素周围空白 | 扣 3-5 分 | 点击可能无效 |
| 多选容器元素 | 整个容器而非目标元素 | 扣 5-10 分 | 操作不精准 |

### 修正建议
1. 缩小框选范围，只框选目标元素
2. 对于按钮，使用 `button` 而非 `div` 选择器
3. 在预览中确认框选区域是否只包含目标元素

---

## 错误类型 7：父子层级错误

### 错误描述
框选了父容器元素而非具体的子元素，或者框选了子元素但实际需要操作父元素。

### 示例

```
DOM 结构：
<div class="form-group">                    <- 父容器
  <label>用户名</label>
  <input id="username" type="text">         <- 目标子元素
</div>

正确：框选 input#username
错误：框选 div.form-group（父容器）
```

### 扣分标准

| 层级错误 | 说明 | 扣分 | 功能影响 |
|----------|------|------|----------|
| 选父不选子 | 选中了父容器 | 扣 8-12 分 | 操作无法精准定位 |
| 选子不选父 | 选中了子元素但需操作容器 | 扣 6-10 分 | 部分功能可能失效 |
| 层级太深 | 选中了深层嵌套元素 | 扣 5-8 分 | 选择器脆弱易失效 |

### 修正建议
1. 理解 DOM 层级结构，确认目标元素在树中的位置
2. 优先选择最具体的子元素
3. 使用 `>` 子选择器提高精度：`div.form-group > input`
4. 在浏览器开发者工具中验证选择器路径

---

## 错误类型 8：动态元素

### 错误描述
框选了页面加载后动态变化或消失的元素，导致后续操作无法定位。

### 示例

```
场景：
1. 页面加载时框选了某个元素
2. 页面执行了 AJAX 刷新
3. 原框选元素被替换/重新渲染
4. 代码仍然引用旧的元素引用 -> 操作失败

典型动态元素：
- 下拉菜单选项（点击后才生成）
- 弹窗内容（打开后才存在）
- 翻页后的列表数据（AJAX 加载）
- 计时器刷新的元素
```

### 扣分标准

| 动态类型 | 说明 | 扣分 | 功能影响 |
|----------|------|------|----------|
| 动态加载 | 元素在操作时未加载完成 | 扣 5-8 分 | 操作延迟或失败 |
| 动态刷新 | 页面刷新后元素引用失效 | 扣 8-12 分 | 需要重新定位 |
| 动态消失 | 元素在其他操作后消失 | 扣 10-15 分 | 后续操作完全失败 |

### 修正建议
1. 动态元素不要提前框选，在需要操作时再拾取
2. 使用 `wait_element()` 等待元素加载完毕
3. 每次操作前重新查找元素而非缓存引用
4. 对于 AJAX 加载的内容，增加适当的等待时间
5. 使用更稳定的选择器（ID > class > 标签 > 位置）

## 框选自检步骤

每次完成框选后，按照以下步骤自检：

```
Step 1: 预览检查
  在 RPA Studio 中预览框选区域
  确认覆盖了正确的目标元素

Step 2: 类型确认
  确认选中的元素类型与预期一致
  检查 CSS/XPath 选择器是否正确

Step 3: 范围确认
  确认框选范围不长不大不小
  用开发者工具测量元素实际尺寸

Step 4: 功能测试
  运行一次功能操作（点击/输入）
  确认操作结果符合预期

Step 5: 容错测试
  多次运行同一操作
  确认每次结果一致
```
