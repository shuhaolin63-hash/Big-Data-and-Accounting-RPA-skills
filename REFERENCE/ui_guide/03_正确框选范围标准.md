# 正确框选范围标准

本文档定义各元素类型的标准框选范围，以及框选精度的评分规则。

---

## 一、各元素类型标准框选范围

### 1.1 输入框类（input）

| 元素类型 | 标准框选范围 | 图示说明 |
|----------|-------------|---------|
| 单行文本输入框 | 选中 input 元素自身，从左边框到右边框，上边框到下边框 | 紧贴 input 元素四个边 |
| 密码输入框 | 选中 input[type="password"] 元素自身 | 与文本输入框相同，但注意 type 不同 |
| 数字输入框 | 选中 input[type="number"] 元素自身 | 同上 |
| 多行文本域 | 选中 textarea 元素自身 | 同上 |

**框选标准**：
- 不包含 label 标签
- 不包含外层 div 容器的 padding 区域
- 不包含相邻的其他 input 元素
- 精确到 input 元素本身的 border 范围

### 1.2 按钮类（button / a）

| 元素类型 | 标准框选范围 | 图示说明 |
|----------|-------------|---------|
| 提交按钮 | 选中 button 元素自身（含文字和边框） | 紧贴按钮四个边 |
| 链接按钮 | 选中 a 元素自身 | 同上 |
| 图标按钮 | 选中 img 或 i/span 图标元素 | 以图标可视区域为准 |

**框选标准**：
- 选中按钮可点击区域（通常在 border 范围内）
- 多个相邻按钮时确保选中的是目标按钮
- 注意区分主按钮（高亮）和次要按钮（普通样式）

### 1.3 表格数据区域（Excel）

| 数据量 | 标准框选范围 | 说明 |
|--------|-------------|------|
| 少量数据（1-10 行） | 精确选择数据行区域（如 A1:F10） | 包含表头 + 所有数据行 |
| 中等数据（10-50 行） | 按实际数据行选择（如 A1:F30） | 同上，不包含空行 |
| 大量数据（50+ 行） | 按实际数据选择或使用动态范围 | 建议用代码检测最后一行 |

**框选标准**：
- 必须包含表头行（第一行）
- 必须包含所有数据行（不遗漏）
- 不能包含数据区域外的空白行
- 列范围必须包含所有需要的字段

### 1.4 图片类（img）

| 元素类型 | 标准框选范围 | 说明 |
|----------|-------------|------|
| 静态图片 | 选中 img 元素可视区域 | 从图片左上角到右下角 |
| 图标 | 选中图标的可视边界 | 紧贴图标边缘 |

### 1.5 下拉框类（select）

| 元素类型 | 标准框选范围 | 说明 |
|----------|-------------|------|
| 原生 select | 选中 select 元素自身 | 紧贴 select 边框 |
| 自定义下拉框 | 选中模拟下拉框的容器元素 | 需要根据具体实现确定 |

### 1.6 复选框/单选框类

| 元素类型 | 标准框选范围 | 说明 |
|----------|-------------|------|
| checkbox | 选中 input[type="checkbox"] 元素 | 或选中的 label（如果包含 checkbox） |
| radio | 选中 input[type="radio"] 元素 | 同上 |

## 二、框选精度评分标准

### 2.1 评分等级

| 等级 | 分数 | 标准 |
|------|------|------|
| 精准 | 26-30 分 | 框选范围完全正确，偏移 < 2px |
| 良好 | 20-25 分 | 框选范围基本正确，偏移 2-5px |
| 合格 | 14-19 分 | 框选范围可接受，偏移 5-15px |
| 偏差大 | 6-13 分 | 框选范围偏差较大，偏移 15-30px |
| 不合格 | 0-5 分 | 框选范围严重错误，偏移 > 30px 或选错元素 |

### 2.2 各等级扣分规则

| 等级 | 每处扣分 | 若 5 处框选均为此等级 |
|------|---------|---------------------|
| 精准 | 不扣分 | 30 分（满分） |
| 良好 | 扣 1-2 分/处 | 20-25 分 |
| 合格 | 扣 2-4 分/处 | 10-19 分 |
| 偏差大 | 扣 4-6 分/处 | 0-13 分 |
| 不合格 | 扣 6-10 分/处 | 0 分 |

### 2.3 特殊情况扣分

| 情况 | 额外扣分 | 说明 |
|------|---------|------|
| 选错元素类型 | 扣 15 分/处 | 如把 button 框选成 div |
| 漏选必需元素 | 扣 10-20 分/处 | 遗漏了某个必须框选的元素 |
| 多选无关元素 | 扣 5-10 分/处 | 框选了不需要操作的元素 |
| 选到动态元素 | 扣 5-8 分/处 | 框选了会变化/消失的元素 |
| 选到父子层级错误 | 扣 8-12 分/处 | 框选了父容器而非目标子元素 |

## 三、框选精度检测方法

### 3.1 视觉检测

```
方法：通过 RPA Studio 预览功能查看框选区域
检查内容：
  1. 框选区域是否覆盖目标元素
  2. 框选区域是否超出目标元素过多
  3. 框选区域是否偏离目标元素
  4. 是否包含了不需要的相邻元素
```

### 3.2 代码检测

```python
def check_selection_accuracy(selector, expected_element_info):
    """
    检测框选精度
    selector: 选中的元素选择器
    expected_element_info: 目标元素预期信息
    """
    # 1. 验证元素是否存在
    element = rpa4.find_element(selector)
    if element is None:
        return {"status": "error", "msg": "元素不存在"}

    # 2. 获取元素位置和大小
    location = element.location  # {'x': ..., 'y': ...}
    size = element.size          # {'width': ..., 'height': ...}

    # 3. 与预期值比较
    expected_location = expected_element_info.get("location")
    expected_size = expected_element_info.get("size")

    if expected_location and expected_size:
        offset_x = abs(location['x'] - expected_location['x'])
        offset_y = abs(location['y'] - expected_location['y'])

        if offset_x < 2 and offset_y < 2:
            return {"status": "perfect", "score": 30}
        elif offset_x < 5 and offset_y < 5:
            return {"status": "good", "score": 25}
        elif offset_x < 15 and offset_y < 15:
            return {"status": "acceptable", "score": 18}
        else:
            return {"status": "poor", "score": 10}

    return {"status": "unknown", "score": 15}
```

## 四、框选检查清单

每一处框选完成后，对照以下清单检查：

### 4.1 基础检查

- [ ] 目标元素是否被完全框选（没有遗漏部分）
- [ ] 框选范围是否超出目标元素（多余面积 < 10%）
- [ ] 框选范围是否偏移（偏移 < 5px）
- [ ] 框选的是正确元素类型
- [ ] 框选区域在页面加载完成后才进行

### 4.2 功能性检查

- [ ] 框选后功能操作是否能正常执行（点击/输入/读取）
- [ ] 操作结果是否符合预期
- [ ] 多次运行是否有一致的表现
- [ ] 页面刷新/重新加载后框选是否仍然有效

### 4.3 特殊场景检查

- [ ] 当前框选是否依赖其他框选的执行顺序
- [ ] 框选元素是否为动态加载（需要额外等待）
- [ ] 框选元素是否在 iframe 中（需要先切换 iframe）
- [ ] 框选元素是否在新打开的窗口/标签页中
