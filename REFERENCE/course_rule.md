# RPA 课程作业考核规则

本文档汇总了 RPA 课程作业的提交规范、查重规则、评分标准和常见违规行为，供老师和学生参考。

---

## 一、作业提交规范

### 1.1 提交方式

| 项目 | 说明 |
|------|------|
| 提交平台 | 学校指定的在线作业提交系统 |
| 提交入口 | 登录学生平台 -> 课程作业 -> 选择对应作业 -> 上传文件 |
| 文件格式 | 提交作业工程文件夹的压缩包（.zip 或 .rar） |
| 命名规范 | `{学号}_{姓名}_{作业名称}_v{版本号}.zip`，例如：`2024001_张三_发票查验机器人_v1.0.zip` |

### 1.2 提交时间

| 时间节点 | 说明 |
|----------|------|
| 截止时间 | 以课程通知为准，通常为作业发布后 1-2 周内 |
| 延迟提交 | 延迟提交按每延迟 1 天扣减总分 10%，最多延迟 3 天 |
| 逾期不交 | 超过最迟提交时间仍未提交，作业按 0 分处理 |

### 1.3 格式要求

| 要求项 | 标准 |
|--------|------|
| 文件完整性 | 必须包含所有 .py / .flow.json / .variable.json / .ctrl.json 文件 |
| 依赖组件 | dependencies/ 文件夹必须完整，含 _global.rpaproj 和 app.rpax |
| 工程结构 | 工程必须能通过 RPA 平台加载并正常运行 |
| 数据表格 | 配套的 .xls 业务数据表必须随工程提交 |
| 运行日志 | 部分作业要求附带运行截图或日志记录 |

## 二、查重规则说明

### 2.1 查重范围

| 查重对象 | 说明 |
|----------|------|
| 源代码文件 | 所有 .py 文件进行代码相似度比对 |
| 流程定义文件 | .flow.json 文件结构和参数比对 |
| 变量配置文件 | .variable.json、.ctrl.json 中的配置信息比对 |
| 选择器数据 | UI 元素选择器（CSS/XPath）路径比对 |

### 2.2 查重阈值

| 相似度等级 | 相似度范围 | 处理方式 |
|-----------|-----------|----------|
| 独立完成 | < 50% | 正常评分 |
| 轻微雷同 | 50% - 65% | 扣减 20% 分数 |
| 明显雷同 | 65% - 80% | 扣减 50% 分数，约谈学生 |
| 高度雷同 | 80% - 95% | 作业记 0 分，通报批评 |
| 完全抄袭 | > 95% | 作业记 0 分，按考试作弊处理 |

### 2.3 查重避坑指南

| 风险点 | 风险等级 | 说明 |
|--------|---------|------|
| 直接复制他人代码 | 高危 | 相似度超过 95% 直接 0 分 |
| 仅修改变量名 | 高危 | 代码结构相同仍会被判定为雷同 |
| 仅修改注释内容 | 高危 | 注释修改不影响代码相似度检测 |
| 共用同一份 ID | 高危 | 系统会检测 UserID 与机器绑定关系 |
| 复用框架代码 | 中危 | 标准框架部分可复用，业务逻辑必须独立完成 |
| 改写部分代码 | 低危 | 修改核心逻辑和框选区域可降低相似度 |

### 2.4 查重处理流程

```
检测到高度相似
      │
      ▼
系统自动标记
      │
      ▼
教师人工复核
      │
  ┌───┴────┐
  │ 确认抄袭 │   │ 误判(撤回标记)
  └───┬────┘   └──────┘
      │
      ▼
按规则处理（0分/通报/作弊）
```

## 三、评分标准总览

总分 100 分，按以下 5 个维度评分：

### 3.1 评分维度

| 评分维度 | 满分 | 说明 |
|----------|------|------|
| 框选精度 | 30 分 | UI 元素选择器定位准确性、框选范围精度 |
| 参数规范 | 25 分 | 超时设置、等待时间、路径配置等工作参数规范性 |
| ID 合规性 | 25 分 | 是否使用个人 ID、是否存在硬编码 ID、ID 注入方式 |
| 文件输出格式 | 20 分 | 输出文件路径、格式、数据完整性和命名规范 |

### 3.2 各维度评分细则

#### 框选精度（30 分）

| 得分档 | 标准 | 典型表现 |
|--------|------|----------|
| 26-30 分（优秀） | 所有框选精准、无偏移 | 元素选择器精确匹配目标，框选范围恰到好处 |
| 20-25 分（良好） | 大部分框选准确，少量轻微偏移 | 1-2 处框选偏移 < 10px，不影响功能 |
| 14-19 分（合格） | 框选基本正确，部分偏移较大 | 3-5 处偏移 < 20px，功能基本正常 |
| 0-13 分（不合格） | 框选偏差严重，影响功能运行 | 多处偏移 > 20px 或选错元素类型 |

#### 参数规范（25 分）

| 得分档 | 标准 | 典型表现 |
|--------|------|----------|
| 21-25 分（优秀） | 所有参数设置合理 | timeout、间隔时间等参数与操作场景匹配 |
| 15-20 分（良好） | 少量参数不合理 | 1-2 处 timeout 过长或过短 |
| 9-14 分（合格） | 部分参数设置不当 | 等待时间设置严重不符，但可通过手动调整 |
| 0-8 分（不合格） | 参数设置严重错误 | timeout 为 0 或负值，路径不存在等 |

#### ID 合规性（25 分）

| 得分档 | 标准 | 典型表现 |
|--------|------|----------|
| 25 分（满分） | 使用个人 ID、无硬编码、动态注入 | 通过 runtime_id_input.py 注入，内存中运行 |
| 15-24 分（部分扣分） | 存在硬编码 ID 但已替换 | 代码中发现他人 ID 但有清理记录 |
| 0 分（不合格） | 使用他人 ID | 检测到 Base64 编码的他人 UserID/TaskID |

#### 文件输出格式（20 分）

| 得分档 | 标准 | 典型表现 |
|--------|------|----------|
| 17-20 分（优秀） | 输出完整、格式正确、命名规范 | 文件路径正确，数据完整，格式符合要求 |
| 12-16 分（良好） | 输出存在少量问题 | 命名略有偏差，不影响数据完整性 |
| 6-11 分（合格） | 输出基本可用 | 路径正确但数据不完整或格式有偏差 |
| 0-5 分（不合格） | 输出文件缺失或乱码 | 文件不存在、路径错误或数据乱码 |

## 四、常见违规行为及后果

| # | 违规行为 | 违规等级 | 扣分/处罚 | 说明 |
|---|----------|---------|-----------|------|
| 1 | 使用他人 UserID 提交作业 | 严重 | 0 分处理 | 系统检测 ID 与机器绑定关系，使用他人 ID 直接判定为 0 分 |
| 2 | 代码中硬编码他人 ID（Base64） | 严重 | 0 分处理 | 就算清除不彻底也会被系统标记为抄袭 |
| 3 | 完全复制他人代码未做修改 | 严重 | 0 分 + 通报 | 查重率 > 95%，按考试作弊处理 |
| 4 | 代码大量雷同（80-95%） | 较重 | 0 分 | 查重触发，记为雷同 |
| 5 | 代码明显雷同（65-80%） | 较重 | 扣 50% | 需约谈说明 |
| 6 | 提交文件不完整 | 中等 | 扣 20-30 分 | 缺少 .py / dependencies 等关键文件 |
| 7 | 工程无法正常运行 | 中等 | 扣 15-25 分 | 代码有语法错误或运行时出错 |
| 8 | 未提交 ID 配置文件 | 中等 | 扣 10-15 分 | 未附带 user_id_config.yaml |
| 9 | 输出文件路径错误 | 轻微 | 扣 5-10 分 | 输出到错误目录 |
| 10 | 输出数据不完整 | 轻微 | 扣 5-10 分 | 数据缺失行/列 |
| 11 | 命名不规范 | 轻微 | 扣 3-5 分 | 压缩包命名不符合规范 |
| 12 | 超时导致运行中断 | 轻微 | 扣 3-5 分 | 未设置合理的 timeout 值 |

## 五、作业提交检查清单

提交作业前，请逐项检查：

### 5.1 必须检查项

- [ ] 所有硬编码 ID 已清除（运行 `id_safe_refactor.py` 确认）
- [ ] UserID/TaskID 已填写到 `user_id_config.yaml`
- [ ] 代码中所有路径已替换为本机实际路径
- [ ] 工程能完整运行一遍不出错
- [ ] 输出文件路径正确、格式符合要求
- [ ] dependencies/ 组件完整
- [ ] 压缩包命名规范

### 5.2 推荐检查项

- [ ] 查重率自查（如有查重工具）
- [ ] 运行截图作为提交附件
- [ ] 框选操作记录填写完整（`select_record.md`）
- [ ] 注释中的 TODO 标记已处理
- [ ] 未使用的 import 已清理
