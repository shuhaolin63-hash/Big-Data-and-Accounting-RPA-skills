# ⚠️ 风控应急处理手册

> 当作业提交被风控拦截、ID 校验失败、账号异常时的**紧急处理指南**

---

## 一、风控等级判定

| 等级 | 表现 | 紧急程度 |
|------|------|----------|
| 🟢 **低风险** | 提示「ID 格式异常」「TaskID 不匹配」 | 可自行处理 |
| 🟡 **中风险** | 提示「账号存在多设备登录」「提交IP异常」 | 需联系老师 |
| 🔴 **高风险** | 账号被锁定、作业被标记为抄袭、代码被判定为舞弊 | 立即停用+申诉 |

---

## 二、常见风控场景及处理

### 场景1：提交时提示「UserID 不合法」

**原因**：代码中残留了他人的 Base64 编码 ID，平台检测到多 UserID 冲突。

**应急步骤**：

```
Step1 停止提交，不要重复尝试
Step2 运行破障Agent：调用 id_safe_refactor.py 全盘扫描
Step3 检查以下文件是否还有残留：
  - rzgc.variable.json → 解码后查看 auth/loginer/token 字段
  - variable.json → 检查 defaultValue 中的默认 ID
  - .py 文件 → 搜索 "user_id"、"task_id"、"token" 赋值语句
Step4 确认全部清除后，重新注入自己的 ID
Step5 再次提交
```

**预防**：每次复用他人代码前，必须先使用 `id_safe_refactor.py` 清洗。

---

### 场景2：提示「TaskID 与作业不匹配」

**原因**：使用的 TaskID 不是老师本次发布的作业对应 ID。

**应急步骤**：

```
Step1 核对老师发布的作业通知，确认正确的 TaskID
Step2 打开 USER_WORKSPACE/user_id_config.yaml
Step3 修改 task_id 字段为正确的值
Step4 重新运行 runtime_id_input.py 注入新 TaskID
Step5 重新生成作业并提交
```

**预防**：不同作业的 TaskID 不同，每次提交前务必核对。

---

### 场景3：提示「代码疑似抄袭」

**原因**：两份作业的代码相似度过高，框选区域完全一致。

**应急步骤**：

```
Step1 不要直接提交
Step2 检查自己的框选记录 USER_WORKSPACE/select_record.md
Step3 与同学对比框选区域是否有差异
Step4 修改至少 3 个非关键框选区域的范围（偏移 5-10px）
Step5 调整参数：timeout 值、等待时间、模拟方式
Step6 重新生成作业并对比是否与之前不同
```

**预防**：框选时故意留微小偏移，不要和同学框得完全一样。每次框选记录自动保存，便于比对。

---

### 场景4：提示「账号异地登录」「IP 异常」

**原因**：在非本机运行，或短时间内多次切换账号。

**应急步骤**：

```
Step1 立即停止当前操作
Step2 确认本机是否为学生版授权设备
Step3 如使用 VPN / 代理，先关闭
Step4 联系老师说明情况：提供本机 IP + 登录时间
Step5 等待平台解封（通常 24 小时内自动恢复）
```

**预防**：始终在本机学生版 RPA 环境中运行，不要跨设备提交。

---

## 三、紧急联系方式

| 情况 | 联系对象 | 需要提供的信息 |
|------|----------|---------------|
| 账号被锁 | 课程老师 | 学号 + UserID + 截图 |
| 作业被退回 | 课程老师 | TaskID + 提交时间 + 退回原因截图 |
| 系统异常 | 平台技术支持 | UserID + 错误码 + 日志文件 |

---

## 四、作业提交前最终检查清单

每次提交前，逐项确认：

- [ ] 代码中无他人的 Base64 编码 ID（运行 `id_safe_refactor.py --check`）
- [ ] `user_id_config.yaml` 填的是你本人的 UserID 和 TaskID
- [ ] 框选记录 `select_record.md` 中有完整的操作记录
- [ ] 所有路径已替换为本机路径
- [ ] 业务表格数据完整可读
- [ ] 作业能正常运行一次（建议先本地跑通再提交）

---

## 五、账号被锁后的恢复流程

```
发现账号被锁
     │
     ▼
停止所有操作，不要重复登录
     │
     ▼
截图保存错误提示信息
     │
     ▼
联系课程老师说明情况
     │
     ▼
等待平台解封（通常 24h 内）
     │
     ▼
解封后先运行 id_safe_refactor.py 检查代码
     │
     ▼
确认无误后重新注入个人 ID
     │
     ▼
重新提交作业
```

---

> **记住**：风控不可怕，可怕的是被锁了还不知道原因。每次操作前先检查 ID，省去 80% 的风控问题。
