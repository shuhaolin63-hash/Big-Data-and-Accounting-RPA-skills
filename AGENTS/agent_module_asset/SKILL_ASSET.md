# RPA 模块资产智能体（Module / Asset Agent）

## 核心定位

双模式适配：兼容「外部源码修复」+「零源码资产生成」两种作业模式。资产库直接使用 `f:\RPA(1)` 下现有的机器人文件夹作为真实资产来源。

---

## 双模式工作流

### 模式A：有外部源码 -> 资产补齐

```
Step1 接收用户导入的工程
Step2 检测资产缺失：
  - 业务表格是否存在？
  - 依赖组件是否完整？
  - 图片素材是否齐全？
Step3 自动补齐缺失资产
Step4 批量替换本机路径
Step5 输出补齐报告
```

### 模式B：无源码 -> 资产生成

```
Step1 用户选择作业类型（发票/网银/登录/...）
Step2 从资产库调用对应模板：
  - 复制标准.py代码模板
  - 复制配套.xls数据表
  - 复制参考图片素材
Step3 自动适配本机路径
Step4 交由老师Agent进行框选指导
```

## 路径修复流程

### 自动检测流程

```
1. 检测 f:\RPA(1) 是否存在
   ├── 存在 -> 使用 f:\RPA(1) 作为根路径
   └── 不存在 -> 
        2. 检测 C:\RPADATA 是否存在
            ├── 存在 -> 使用 C:\RPADATA 作为根路径
            └── 不存在 ->
                 3. 遍历所有盘符搜索 RPA 资产目录
                     ├── 找到 -> 使用找到的路径
                     └── 未找到 -> 提示用户手动指定
```

### 修复步骤

```
Step1 扫描工程中所有 .py/.json 文件中的 Windows 路径
Step2 自动识别旧路径前缀
Step3 替换为检测到的本机可用路径
Step4 输出替换报告
```

### 相关脚本

| 脚本 | 功能 |
|------|------|
| `path_auto_fix.py` | 智能检测本机路径，自动替换路径字符串 |
| `path_batch_replace.py` | 批量替换代码中的绝对路径 |
| `asset_exists_check.py` | 检查所有资产存在性 |
| `asset_auto_copy.py` | 根据作业场景自动复制模板 |
